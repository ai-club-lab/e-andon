# Cost & Context Efficiency

トークン消費と作業効率を揃えるための共通ルール（個人設定からハッカソン汎用部分だけ抽出）。

## コンテキスト管理
- 必要なファイルだけ読む。ディレクトリ丸ごと読まない。
- まず grep/glob で当ててから対象を読む。
- 長い出力は要約。全文貼り付けを避ける。
- タスクを切り替えたら不要なコンテキストを閉じる（`/clear`）。

## モデル使い分け（コスト目線）
- 既定コーディング: **Opus 4.8**（`claude-opus-4-8`）。
- 浅い参照・雑談: **Sonnet 4.6**。サブエージェント量産・並列リサーチ: **Haiku 4.5**（frontmatter でピン）。
- Effort は既定 medium。アーキ/多ファイル/セキュリティのときだけ high。
- 自動・スケジュール実行はフルモデルIDを固定（alias 不可）。

## 効率的な進め方
- 3ファイル超のタスクは Plan モード。
- 関連する編集はまとめて行う。
- 単純検索は grep/glob を直接使い、探索を広げすぎない。

## レスポンス
- 簡潔に。冗長な説明を避ける。
- 変更は diff / 該当箇所で示す。全文表示しない。
- 明らかな確認は省く（聞かれない限り）。

## MCP
- 既定は無効。プロジェクトで必要なものだけ有効化。
- 目安: 有効MCP <10 / アクティブツール <80。組込ツールで足りるなら MCP を使わない。
