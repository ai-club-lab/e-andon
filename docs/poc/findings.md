# PoC: 決定論CVによるライン異常検知（チョコ停）

対象素材: `video/factory_01.mov`（1014x634 / 24fps / 10秒）
検証コード: [`detect_v2.py`](detect_v2.py)（OpenCV, ローカル `.venv`）

## 結論

**決定論CVで異常が実データ上クリーンに検知できることを確認。** テーマの技術的成立性が取れた。
ハイブリッド二段検知の1段目（CV）は固い。

## 手法

1. ベルト帯を ROI（y=250..430）に限定
2. ROI内をグレースケール→Otsu二値化→モルフォロジで部品をブロブ化
3. 各部品の重心 `(cx,cy)` と回転角（`minAreaRect`）を算出、面積・アスペクトでフィルタ
4. フレーム内の中央値を基準に、以下3シグナルで異常判定
   - `cy オフセット`：行の中心線からの縦ズレ
   - `回転角の逸脱`：整列向きからのズレ
   - `間隔ギャップ`：隣接間隔 > 1.5×中央値 → 部品が定位置を外れた（欠け）

## 測定結果

| 項目 | 値 |
|---|---|
| 正常ラインのばらつき | cy σ≈1px / 角度 σ≈1° / 間隔ほぼ一定（S/N 良好） |
| 異常の実体 | 部品1個が **約18px 上方へ変位**。frame **139–240**（後半4秒）を右→左に通過する単一イベント |
| 誤検知 | frame 0–138 は全クリーン（false positive ゼロ） |
| 閾値マージン | 雑音<2px に対し異常18px → 閾値 10–12px で余裕 |

- 当初想定の「回転異常」は本素材には存在せず（初見はJPEG劣化の見間違い）、実異常は**縦ズレ1種**。
  角度検知ロジックは実装済み・安定動作するため、将来の回転異常にもそのまま適用可能。
- 変位部品が上部レールと融合して分割から漏れるケース（frame216）は、`間隔ギャップ`シグナルが拾う。

## 本実装への申し送り

1. **時系列トラッキング必須**：同一異常を100フレーム個別発火させず、変位部品を追跡して
   **1異常イベント（part_id / 開始-終了 / ピークオフセット）** に集約してからエージェントへ渡す。
2. **2シグナル統合**：`offset` を主、融合時は `gap` をバックアップに。
3. **Gemini二段確認**は境界帯（offset 8–12px）のみに限定し、呼び出し・コストを最小化。
